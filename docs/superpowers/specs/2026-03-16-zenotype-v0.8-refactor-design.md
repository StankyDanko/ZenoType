# ZenoType v0.8.0 — Component Extraction, TypeScript Migration & Remote AI

**Date:** 2026-03-16
**Status:** Approved
**Scope:** Full architectural refactor of ZenoType from monolithic JSX to modular TypeScript, plus server-side Ollama exposure for live demo.

---

## 1. Goals

1. Extract the 2135-line `App.jsx` monolith into focused TypeScript components, hooks, and constants
2. Migrate from JSX to TypeScript (`.tsx`/`.ts`) with strict mode
3. Make the Ollama endpoint configurable so the GitHub Pages demo connects to a remote server
4. Set up the Reliable VPS to serve Ollama inference for the live demo
5. Bump version to v0.8.0

## 2. Non-Goals

- Adding a state management library (Zustand, Redux, etc.)
- Adding a test suite
- Changing any visual styling, animations, or game mechanics
- TypeScript strict-null or exhaustive type coverage on day one — pragmatic types, not perfect types

## 4. Configurable Ollama Endpoint

### 4.1 Config Module

```typescript
// src/config.ts
const OLLAMA_ENDPOINTS = {
  local: 'http://localhost:11434',
  remote: 'https://zenotype-api.southernsky.cloud',
} as const;

export const OLLAMA_BASE_URL =
  import.meta.env.VITE_OLLAMA_URL || OLLAMA_ENDPOINTS.remote;
```

### 4.2 Resolution Priority

| Context | Env Var | Resolved URL |
|---------|---------|-------------|
| GitHub Pages (prod) | none | `https://zenotype-api.southernsky.cloud` |
| Local dev with local Ollama | `VITE_OLLAMA_URL=http://localhost:11434` in `.env.local` | `http://localhost:11434` |
| Local dev, remote AI | none | `https://zenotype-api.southernsky.cloud` |

### 4.3 Boot Sequence & URL Migration

Unchanged behavior: fetch `${OLLAMA_BASE_URL}/api/tags` → if models found, enable AI mode + fetch topics. If fetch fails, gracefully fall back to offline mode with static sentence pools.

**All 4 hardcoded `localhost:11434` references must be updated:**
1. Boot sequence model detection (`/api/tags`)
2. Topic generation (`/api/generate` in `fetchTopicsFromOllama`)
3. Text generation (`/api/generate` in `fetchOllamaWords`)
4. Scholar insight generation (`/api/generate` in `fetchScholarInsight`)

Each becomes: `` `${OLLAMA_BASE_URL}/api/generate` `` (or `/api/tags` for boot).

## 5. Component Architecture

### 5.1 Type Definitions (`src/types.ts`)

Shared types used across the codebase:

```typescript
export type Difficulty = 'adaptive' | 'standard' | 'code' | 'syntax' | 'scripture';
export type WordType = 'standard' | 'code' | 'syntax' | 'transformer' | 'ollama' | 'verse-num';
export type AppPhase = 'booting' | 'topic-select' | 'playing';
export type GameStatus = 'idle' | 'typing' | 'paused';

export interface Word {
  type: WordType;
  text: string;
}

export interface KeyStat {
  hits: number;
  misses: number;
}

export interface BibleProgress {
  bookIdx: number;
  chapter: number;
}

export interface CuratedPassage {
  label: string;
  book: string;
  chap: number;
}

export interface ThemeConfig {
  text: string;       // Tailwind classes (e.g., "text-cyan-400")
  bg: string;         // Tailwind classes (e.g., "bg-cyan-400")
  glow: string;       // Tailwind shadow classes
  rawColor: string;   // Hex value for inline CSS (e.g., "#22d3ee")
  rawGlow: string;    // RGBA value for CSS animation (e.g., "rgba(34,211,238,1)")
}
```

### 5.2 Constants (`src/constants/`)

| File | Contents |
|------|----------|
| `keyboard.ts` | `KEYBOARD_ROWS`, `FINGER_MAP`, `SHIFT_CHARS` |
| `sentences.ts` | `SENTENCES_STANDARD`, `SENTENCES_CODE`, `SENTENCES_SYNTAX`, `HALLUCINATIONS`, `FALLBACK_TOPIC_POOL`, `getRandomFallbackTopics()`, `generateWords()` |
| `bible.ts` | `BIBLE_BOOKS`, `CHAPTER_COUNTS`, `CURATED_PASSAGES` |
| `theme.ts` | `THEME_CLASSES`, `getThemeForTPM()` (the heat engine color ladder) |

### 5.3 Hooks (`src/hooks/`)

#### `useOllama.ts`
- **Owns:** `ollamaEnabled`, `isGenerating`, `isGeneratingTopics`, `ollamaError`, `availableModels`, `selectedModel`, `dynamicSuggestions`, `activeThread`, `flowMilestone`
- **Internal refs:** `generateAbortControllerRef`, `topicAbortControllerRef`, `threadHistoryRef`
- **Exposes:** `fetchTopicsFromOllama()`, `fetchOllamaWords()`, `setSelectedModel()`, `setOllamaEnabled()`, `setActiveThread()`, `abortAll()`
- **Accepts:** `aggregatedKeyStats`, `tpm`, `difficulty`, `tScore` as parameters/config
- **Note:** `flowMilestone` lives here because `fetchOllamaWords` is the only writer (the flow-state check at generation time). `activeThread` lives here because it's set when a topic is selected and read during generation. Both are exposed as read-only values for the UI.

#### `useScripture.ts`
- **Owns:** `bibleProgress`, `scholarInsight`, `isFetchingScholar`, `customBibleSearch`, `bibleApiError`, `showVerseNumbers`, `bibleVersion`
- **Exposes:** `loadBibleChapter()`, `advanceBibleChapter()`, `parseBibleSearch()`, `fetchScholarInsight()`
- **Accepts:** `ollamaEnabled`, `selectedModel` (reads `OLLAMA_BASE_URL` from config module directly)

#### `useGameEngine.ts`
- **Owns:** `status`, `timeMs`, `totalChars`, `tScore`, `highScore`, `scrollOffset`
- **Internal refs:** `lastKeystrokeRef`, `cumulativeTimeRef`, `historyRef`, `currentStatsRef`
- **Exposes:** `tpm`, `accuracy` (computed), `chartData` (downsampled from `historyRef`), `resetGameState()`, timer management
- **Accepts:** `words`, `activeWordIndex`, `difficulty`, `currentInput`, `wordHistory`
- **Note:** Does NOT own `handleInputChange` — the input handler is in App.tsx because it coordinates across `useKeyStats.recordKeystroke()`, `useScripture.advanceBibleChapter()`, and game state updates. `resetGameState()` zeroes all game state (timeMs, tScore, totalChars, scrollOffset, refs) and is called by the session starters in App.tsx.

#### `useKeyStats.ts`
- **Owns:** `keyStats` (localStorage-backed)
- **Exposes:** `aggregatedKeyStats` (memoized), `recordKeystroke(expectedChar, typedChar)`, `resetStats()`

#### `useHotkeys.ts`
- **Owns:** nothing — dispatches callbacks
- **Accepts:** A callback record with stable refs:
  ```typescript
  interface HotkeyCallbacks {
    onToggleAnalytics: () => void;
    onToggleSettings: () => void;
    onToggleHands: () => void;
    onReset: () => void;
    onVisibleLinesUp: () => void;
    onVisibleLinesDown: () => void;
  }
  ```
- **Note:** Uses a `callbacksRef` pattern internally so the event listener registered with `[]` deps always reads current callbacks without re-registering.

### 5.5 App.tsx (~250 lines)

The orchestrator. Owns:
- `appPhase`, `words`, `activeWordIndex`, `currentInput`, `wordHistory`, `difficulty`, `visibleLines`, `showSettings`, `showAnalytics`, `soundEnabled`, `confirmReset`, `customTopicInput`
- Refs: `inputRef`, `activeWordRef`, `containerRef`

**Session starters (live in App.tsx):**
- `startSessionWithTopic(topic)` — Calls `ollama.abortAll()`, `ollama.setActiveThread(topic)`, `gameEngine.resetGameState()`, resets words/index/input/history, then calls `ollama.fetchOllamaWords(true, ...)`.
- `startSessionOffline(diff)` — Same reset pattern, then sets words via `generateWords()`.
- `handleReset()` — Replaces the old `resetTrigger` pattern. Dispatches to scripture topic-select, `startSessionWithTopic`, or `startSessionOffline` based on `difficulty` and `ollamaEnabled`. Called by `useHotkeys` via `onReset` callback.

**Input handler (lives in App.tsx):**
- `handleInputChange(e)` — Coordinates across hooks: calls `gameEngine` for status/timing, `keyStats.recordKeystroke()` for accuracy, `scripture.advanceBibleChapter()` for chapter completion, plays audio (inline AudioContext chirp when `soundEnabled`).

**Computed values (in App.tsx):**
- `activeFingers` — `useMemo` computing which fingers to highlight, using `FINGER_MAP`, `SHIFT_CHARS`, current word, input, and UI state. Passed to `GuideHands` as a prop.

Composes all hooks and components. All cross-hook coordination goes through App.tsx — hooks never call each other directly.

## 8. Validation

Since there are no automated tests, validation is:

1. `npm run build` succeeds with zero TypeScript errors
2. Manual smoke test:
   - Boot → topic select (AI-generated topics load from remote)
   - Type a paragraph → TPM/accuracy update correctly
   - Switch to Scripture mode → Bible API loads chapter
   - Settings panel → all toggles work
   - Analytics overlay → heatmap renders, chart populates
   - Guide hands → fingers light up correctly
   - Alt+R → returns to topic select
   - Offline mode → disconnect network, verify fallback sentences work
